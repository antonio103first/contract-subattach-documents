# 투자 체크리스트 자동 생성 시스템 (Investment Compliance Check)

## 프로젝트 개요

케이런벤처스에서 **투자계약 전 작성해야 하는 2종의 체크리스트**를 투자계약서와 투자심사보고서로부터 자동 생성하는 시스템.

### 생성 문서
1. **투자계약서 체크리스트 및 의무기재사항확인서** (`.docx`)
2. **준법사항체크리스트** (`.hwpx`)

### 입력 형식 지원
- **DOCX** (Microsoft Word) - 투자계약서, 투심보고서
- **HWP v5.0** (Hangul) - 투자계약서 (olefile PrvText 기반)
- **PDF** - 텍스트 PDF는 직접 추출, 스캔본은 Tesseract OCR

## 아키텍처

```
contract-subattach-documents/
├── generate_checklists.py          # 메인 CLI 스크립트
├── extractors/
│   ├── contract_extractor.py       # 투자계약서 파서 (키워드 기반 조문 탐색)
│   ├── report_extractor.py         # 투자심사보고서 파서 (테이블 자동 탐색)
│   └── pdf_extractor.py            # PDF/OCR 처리
├── generators/
│   ├── docx_generator.py           # DOCX 체크리스트 생성 (python-docx)
│   └── hwpx_generator.py           # HWPX 준법사항체크리스트 생성 (ZIP+XML 치환)
├── (양식) 준법사항체크리스트(안)_2024 IBK혁신 케이런 모빌리티 7호 펀드_251208.hwpx
├── (양식) 투자계약서 체크리스트 및 의무기재사항확인서_2024 IBK 혁신펀드.docx
├── output/                         # 생성 결과물
└── .claude/skills/generate-checklist.md  # Claude Code Skill 정의
```

### 실행 방법

```bash
python generate_checklists.py \
  --contract "투자계약서.docx" \
  --report "투심보고서.docx" \
  --output-dir ./output
```

### 의존성
- `python-docx` - DOCX 읽기/쓰기
- `docxtpl` - Jinja2 기반 DOCX 템플릿 (현재 미사용, 직접 셀 치환 방식)
- `lxml` - XML 파싱
- `olefile` - HWP v5.0 PrvText 추출
- `pymupdf` (fitz) - PDF 텍스트 추출
- `pytesseract` + Tesseract OCR - 스캔 PDF
- `Pillow` - 이미지 처리

## 작성지침 (핵심 규칙)

### 표1. 투자기업
- 업체명/대표이사/사업자등록번호/소재지: 투심보고서에서 추출

### 표2. 투자내용
- **구분 컬럼**: 보통주/상환전환우선주/전환우선주/CB/BW 등 계약서에서 판단
- **투자금액/단가**: 투자계약서와 심사보고서 비교, 불일치 시 주석
- **지분율**: 심사보고서
- **주요 투자조건**: 존속기간, 상환조건, 전환조건, Refixing, Tag-Along, Drag-Along 등
- **동반투자내역**: 투심보고서 **동반투자기관 테이블**에서 투자기관/금액/형태 추출

### 표3. 투자담당자
- 발굴자/심사자/사후관리자: 심사보고서 첫페이지

### 표4. 벤처기업 등 해당여부
- **창업기업**: 심사보고서 설립년도 기준 7년 초과 판단 (자동)
- **벤처기업**: 심사보고서에서 벤처기업인증 확인
- **이노비즈/메인비즈**: 심사보고서에서 해당 인증 확인
- 선택값만 적색(157), 미선택값은 텍스트 제거

### 표5. 준법사항 확인

#### 법령상 투자제한 (통상값)
모두 **"부"** 기본:
- 자기/제3자 이익을 위한 조합 재산 사용
- 투자기업의 상호출자제한기업집단 소속 → **비고란: [확인 필요: 중소기업 여부]** (적+기울임)

- 금융회사등 주식/지분
- 이해관계인 발행/소유 주식 → 비고란: "이해관계인 : {이름}"
- 이해관계인 신용공여
- 조합 명의 제3자 주식 취득/자금 중개
- 투자계약서 외 별도 조건
- 기타 법령 위반

## HWPX 생성 방식

**양식 복사 + 치환 방식**을 채택함.

```python
1. (양식) HWPX 파일을 ZIP으로 읽기
2. Contents/section0.xml 에서 placeholder 치환
3. Contents/header.xml 에 charPr 156(적색+기울임), 157(적색만) 추가
4. Preview/PrvText.txt 도 동일하게 치환
5. 원본 메타데이터(flag_bits) 보존하여 ZIP 재작성
```

### 치환 단계 (순서 중요)
1. 단순 치환 (`㈜AAA`, `000-00-00000` 등)
2. 조건/위약벌 개별 태그 (` - 존속기간 :` 등)
3. 순서 기반 치환 (`OOO` 4회: 대표→발굴→심사→사후관리)
4. 표2 투자유형/금액/단가/지분율 (`>원<`, `>%<` 순서)
5. 표4 적(Y)/부(N) 선택
6. 투자방법 (O) 체크
7. 표5 실제 칼럼 빈 셀 채우기 (colAddr="2")
8. 비고란 적색 주석 (colAddr="3")
9. **담당자 확인 행 전체 적색** (역순 처리 - 앞쪽 행에 영향 X)
10. 텍스트 기반 주석 (발굴경위, TCB등급, 투심위예정일)

### 주요 기술 이슈 해결

**1. Git 바이너리 손상**
- 원인: Git autocrlf가 HWPX 바이너리의 줄바꿈 변환
- 해결: `.gitattributes`에 `*.hwpx binary` 등 지정

**2. ZIP flag_bits 손실**
- 원인: Python `zipfile.writestr()`이 flag_bits(UTF-8 인코딩 플래그)를 0으로 리셋
- 해결: ZIP 바이너리에서 직접 flag_bits 패치 (`_patch_flag_bits()`)

**3. XML 특수문자 이스케이프**
- 원인: `M&A`, `SaaS & Analytics`의 `&` 문자가 미이스케이프되어 invalid XML
- 해결: `_xml_safe()` 함수로 `&` → `&amp;` 변환

**4. 한컴오피스 호환성**
- 양식 HWPX의 원본 바이너리를 최대한 보존
- 수정이 필요한 파일(section0.xml, header.xml, PrvText.txt)만 교체

## 개발 히스토리 (주요 커밋)

### Phase 1: 초기 구현
- 투자 체크리스트 자동 생성 시스템 구현 (파서 + 생성기)
- PDF 스캔본 입력 지원 (Tesseract OCR)
- HWP 파일 입력 지원 (olefile PrvText)

### Phase 2: 양식 대응성 강화
- 샘플3(듀셀), 샘플4(타스글로벌, HWP) 대응
- 다양한 계약서 구조 지원 (제1조/제5조 등)
- 테이블 자동 탐색 (회사개요가 Table 1/2/N 어디든)

### Phase 3: HWPX 생성 방식 전환
- XML 직접 생성 → 양식 복사+치환 방식 (한컴오피스 호환성 확보)
- `.gitattributes` 추가 (바이너리 파일 보호)
- ZIP flag_bits 복원

### Phase 4: 작성지침 반영
- 표5 실제 칼럼 적/부 자동 채움 (colAddr=2 빈 셀)
- 별첨1 주요투자조건 요약서 우선 참조
- 비고란 적색 주석 (charPr 156)
- 담당자 확인 행 전체 적색 (charPr 157, 역순 처리)

모든 샘플이 다음을 통과:
- ✅ XML valid
- ✅ 표1~5 자동 채움
- ✅ 불일치 자동 감지 및 주석
- ✅ 한컴오피스에서 정상 열림

## 알려진 한계

1. **HWP v5.0 입력**: PrvText 스트림만 읽으므로 표 구조는 추출 불가 (텍스트 기반 파싱)
2. **PDF 스캔본**: OCR 정확도에 의존 (80~95%)
3. **계약서 구조 다양성**: 극단적으로 다른 양식은 수동 확인 필요
4. **charPr 속성**: 양식에 정의되지 않은 색상/서식은 사용 불가
